# scrapers模块： 具体页面的交互过程及抓取
- seefar_scraper 只负责seefar店铺信息的交互和获取，以及seefar店铺列表商品信息的获取
- ozon_scraper 只负责商详的信息交互和抓取，不负责逻辑, 最后按要求返回商品的详情信息
- competitor_scraper 只负责跟卖店铺的抓取和交互
- erp_plugin_scraper 只负责erp插件区域的信息抓取
- api_scraper 利用api接口获取数据（待建）

# Business包里是具体的业务逻辑, 包括了规则计算，价格计算等，提供了各种函数可以通过xp_flow注入到各个数据抓取流程中
- 价格计算，包含了计算真实的绿标、黑标价格， 佣金、定价逻辑
- 利润评估，通过输入综合的商品信息并利用**利润计算器**，真实的计算利润
- 店铺评估，通过店铺信息以及商品信息，计算店铺的综合评分，用于店铺筛选

# 流程编排模块
-  xp_flow是整个流程编排，以及业务组件的注入，除了满足整体的流程，也可以将business的逻辑高效的注入到各个数据抓取流程中
