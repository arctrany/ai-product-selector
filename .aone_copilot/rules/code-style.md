# 智能选品系统代码风格规范

本项目遵循以下代码风格规范和约定：

## Python 代码风格

### 命名约定

- 类名使用 PascalCase 命名法
- 函数和变量名使用 snake_case 命名法
- 常量名使用 UPPER_SNAKE_CASE 命名法
- 私有成员使用单下划线前缀

### 注释规范

- 类和函数必须有文档字符串（docstring）
- 复杂逻辑需要添加行内注释
- 使用中文注释，关键点使用emoji标识：
  - 🔧 表示修复或关键调整
  - ✅ 表示成功或完成
  - ⚠️ 表示警告或注意事项
  - ❌ 表示错误或失败

### 代码组织

- 每个模块应有清晰的职责划分
- 遵循单一职责原则
- 使用类型提示增强代码可读性
- 合理使用异常处理机制

### 异常处理

- 明确捕获特定异常类型
- 记录详细的错误日志
- 提供有意义的错误信息

### 性能考虑

- 避免重复创建昂贵的对象
- 合理使用缓存机制
- 控制并发数量避免资源耗尽
- 及时释放资源
